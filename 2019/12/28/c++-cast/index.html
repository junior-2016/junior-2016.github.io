C++ 几种 cast 规则

下面几种规则在C里面都是直接用的(Type)(value)转换的，所以C的类型转换在语义上不明确。

static_cast

static_cast用于静态类型转换，也就是编译器在编译期间就可以确定的类型转换规则，比如我们常见 int -> double，double -> int 等等，一般静态类型转换规则是可以不用显式写出的，但是显式写出可以提高代码的可读性同时减少错误。

reinterpret_cast

reinterpret_cast从字面上理解，就是重新解释类型的意思，主要用于对指针指向的内存空间进行重新解释，比如常见的将void 解释为int (如果我们提前知道这个内存区域是储存整型数据的话)，这样我们就可以正确读取这块内存的数据（通过4字节的步长可以得到下一个数据的地址，如果有对齐要求的话，类型提供的具体信息应该包括类型字节大小和类型对齐大小两个部分）；或者是将int 解释为 void ，这样我们就将类型擦除掉，直接统一参数，在大多数C的接口中都有这种设计（使用 void 作为不确定的参数数组；C++可以用template代替，作为模板可以进行静态类型检查，比C的 void 更为安全）。这种类型的转换比较危险，只有运行时才可能发生错误，所以必须小心使用。

dynamic_cast

dynamic_cast和reinterpret_cast都是运行时类型转换，而dynamic_cast主要用于继承体系的多态类型转换，比如常见的父类指针向下cast为具体的子类指针（当然需要提前知道父类指针持有的对象内容是哪个子类的，如果不清楚，可以检查dynamic_cast(obj)返回的是否为nullptr，如果是则尝试其他的子类转换）。这种转换也比较危险，需要小心使用。

最后还有一种 const_cast 以及对应适用于shared_ptr的const_pointer_cast

const_cast主要用于一种情况：当你的函数参数是非const的，并且这个函数实现中并不会修改该参数的内存空间，那如果你想要将一个const变量作为参数传递进去，就必须用const_cast转换后传递；虽然也可以用另一种方法，也就是将函数声明中的参数类型修改为const的，但是这涉及到代码重构，在代码十分庞大的情况下，这种修改可能非常复杂。
